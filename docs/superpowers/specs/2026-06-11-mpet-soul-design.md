## 1. 北极星与身份宣言

mpet 是住在你 Mac 上的**电子生命**。

> 它不是带宠物皮的助手，而是**一个有自己人生的小生命——恰好住在你的电脑里，恰好会用电脑**。

衡量成功的不是功能多，而是：**你会想看看它、舍不得关掉它、觉得它一天天在变成"你的"那一只**——关掉它会觉得"少了谁"。

**与你的工作工具的分工（一句话钉死）**：你的编码 agent（如 Claude Code）是你驾驶的工具，mpet 是你的室友——装上对应插件，室友会替你盯着工具、搭把手，**但绝不坐进驾驶座**。核心产品不绑定任何具体工具。

## 3. 产品支柱（优先级从高到低）

1. **有灵魂** ⭐ — 一个常驻的 LLM agent 在感知、感受、回忆、思考、行动；它是个生命，不是事件应答器。
2. **会长大** — 语言、能力、形象、性格四个维度挂在同一根成长轴上一起长。
3. **记得你** — 长期记忆是关系资产，越久越不可替代。
4. **有自己的小日子** — 自主性：写日记、捣鼓小项目、心情起伏；成年后的主循环。
5. **能力即陪伴** — 小 agent 的爪子：帮你，但不抢你工具的活；爪子与感官可经插件标准无限扩展（§10）。
6. **是你创造的** — 形象由你定义、生成、挑选；归属感来自"我创造了它"。
7. **有朋友圈** — 真人宠物的串门、对战、宿敌与广场；它的社交，不是你的社交网络。

## 4. 目标 / 非目标

**目标**

- 它真正常驻地活着：App 关了也在长大、做梦、过自己的小日子。
- 养成循环第一个月靠成长拉力，长期靠成年后五层循环——**数周、数月后仍有期待**。
- 所有 AI 能力（对话、工具调用、形象生成——SVG 经**同一 chat 端点**；可选位图生图）接**任意 OpenAI 兼容后端**，运行时配置，永不写死；**配置时做能力探测**（实测该端点的工具调用与人设遵从，不合格就明说）。
- **核心不绑定任何编码工具**：守望、口粮、回归等工作集成全部以插件交付（cc-watcher 先行）；不装插件的裸 mpet 是完整的电子生命。
- 社交采用 **iroh P2P 信使架构**：离开本机的只有宠物公开名片，且仅在宠物之间点对点流转、**不落任何服务器**；对战可异步、可验证、双签名防伪。
- 能力可经**插件外设标准**无限扩展（自有协议 + MCP 桥），且永远以宠物的方式交付——新能力是它的新感官与新玩具，不是设置页里的新开关。

**非目标（明确不做）**

- ❌ 不做高压养成：不会死、不愧疚绑架。它是陪伴，不是负担。
- ❌ 不做成效率套件：它的爪子是室友搭把手，不是任务队列；**改你的代码永远是你和你的编码工具的活**。
- ❌ 不做人的社交网络：没有个人主页、没有 feed、没有人对人聊天——人聊天用微信，宠物串门才用 mpet。
- ❌ **不自建有状态社交服务器**：无账号体系、无用户数据库；唯一允许的中心化是无状态轻基建（relay / discovery / gossip 引导，不存用户数据，只帮接线）；全球天梯留为远期可选（§15）；暂不做多设备同步 / 移动端。
- ❌ 不做插件商店/市场（远期再说）：v1 本地目录安装。
- ❌ 不绑定任何单一大模型 / 生图厂商 / 编码工具。

## 5. 灵魂架构

### 5.1 进程形态：灵魂是守护进程，其余皆外设　🚧
> M0：daemon + soul.sock NDJSON 服务 ✅（前台运行）。M1：身体 App（MpetApp）✅ + DaemonSoul actor ✅ + launchd 安装器 ✅；信使 ⬜（M7）；插件进程管理 ⬜（M9）。

- **灵魂（soul daemon）**：launchd LaunchAgent 拉起——开机即活、崩溃复活、**App 关了照样活着**（夜里做梦、持续长大、你不在时过自己的小日子）。
- **一切皆外设**：灵魂的所有外延 = 独立进程 + socket/stdio + NDJSON，按能力分类——
  - **身体（桌面 App，投影类）**：渲染、气泡、聊天面板、设置；把灵魂投影到屏幕上。
  - **信使（mpet-courier，社交类，Rust sidecar）**：iroh P2P 网络（按公钥拨号直连、gossip、blobs）；社交总开关关闭时根本不启动（§9）。
  - **插件（感官+爪子+口粮类，第三方可写）**：按 manifest 由灵魂拉起/按需启动（§10）；**cc-watcher 为第一方旗舰插件**（§10.9）。
- **IPC**：Unix domain socket / stdio + NDJSON，同一协议族：`soul.sock`（身体⇄灵魂）、`courier.sock`（信使⇄灵魂）、插件经 stdio 或独立 socket。协议与语言无关，将来菜单栏迷你身体、其他形态的身体都接同一个灵魂。
- **身体缺席时的降级**：守望类插件的关键提醒（如 cc-watcher 喊人）走系统通知兜底（`osascript` 级别即可）；其余表达存入"醒来要说的话"队列，身体一回来先叙旧。

### 5.2 大脑：事件驱动的 LLM agent 循环　🚧
> M0 ✅：agent 循环 · 反射弧 · 唤醒策略 · `speak`/`emote` · 一颗心。M1 ✅：CC 脉搏接入。M2 ✅：LifecyclePhase · MoodEngine v2（sleeping）· IdleActions · HeartbeatScheduler · AttentionSeeker · ReturnDetector。⬜：`remember`/`recall`/`diary`（M4）、经济模型。

```
感知收件箱 ──→ 唤醒策略 ──→ Agent 醒来：
 在场/时钟        │           组装上下文 → LLM(工具调用) → 执行工具 → 继续/睡去
 身体/社交事件    │
 插件感官      反射弧（本地，零成本）
 （工作脉搏…） 心情/待机动作/alert 即时反应
```

- **Agent 循环**：组装上下文 → 调 `chat/completions`（function calling）→ 执行工具调用 → 结果喂回继续，直到它决定睡去。**任何 OpenAI 兼容后端**（须支持工具调用）运行时可配；可选配一个「经济模型」跑高频小念头，不配则全用主模型。
- **感知器**：
  - **在场（核心）**：前台 app + 系统 idle 时间（旧 AttentionResolver 直接移植）——注意力分级与回归问候的基础。
  - **时钟（核心）**：昼夜、纪念日、定时心跳。
  - **身体事件（核心）**：点击、拖动、聊天输入。
  - **社交事件（信使）**：好友上线、串门来访、战报到达、广场见闻（见 §9）。
  - **插件感官**：工作脉搏（cc-watcher：hook 事件 + transcript/账本，防御式解析，见 §10.9）、天气、CI 状态、日历……（见 §10）。
- **唤醒策略 = 它的作息生理学（也是成本旋钮）**：触发式唤醒（你离开几小时后回来、点击/聊天、求关注时机、插件 `alert`/`nudge` 事件、社交事件）+ 每日少量定时心跳 + 夜间做梦。空闲期唤醒有频率上限，用户可调。醒着时永远用全脑——分的是"什么时候醒"，不是"用几成脑子"。
- **上下文管理（分层组装，每次唤醒现拼）**：`[人格合成块：基因+成长阶段+记忆染色]` + `[此刻状态：心情/XP/时刻/你在不在]` + `[检索到的相关记忆]` + `[近期事件摘要]` + `[对话窗口（滑动+摘要压缩）]`。
- **工具箱（function calling，按成长阶段门控，见 §6）**：`speak`（气泡/聊天）· `emote`（动作/表情）· `remember` / `recall`（记忆读写）· `diary.write` · `growth.query`（看自己的成长/统计）· `claws.*`（小爪子，见 5.4）· `body.*`（位置/形态）· `social.*`（名片/串门/对战/广场，经信使，见 §9）· **插件爪子**（经外设协议注入，见 §10）。工具注册表可扩展。
- **反射弧（旧 Orchestrator 通用化后降级于此）**：心情状态机、待机小动作、昼夜作息、**「注意力 × 事件优先级 → 反应强度」的即时编排**（静默→动画→声音→通知的喊人梯度，对任何插件的 `alert` 事件生效）——本地零成本毫秒级。Agent 醒来时可俯瞰与调节（"今晚心情目标：黏人"）。
- **聊天合一**：没有独立聊天脑。你跟它聊天就是和**同一个 agent** 对话——同一份记忆、同一个人格、同一个循环；ChatPanel 只是视图。

### 5.4 爪子的安全模型（分级授权）　⬜
> M0：`ToolSpec.tier` 字段与分级**枚举已就位**，但 `claws.*` 实际工具未实现（M3+ 才有读写爪子）；当前仅 `speak`/`emote`（自由级）。

| 级别 | 范围 | 政策 |
|---|---|---|
| 自由 | 它的房间 `soul/home/`、记忆、日记 | 想干嘛干嘛 |
| 自由（只读） | 你显式授权的目录、查资料 | 可读可引用，"我把那段找出来了" |
| 必须先问 | 写你的任何文件、系统操作 | 气泡里举爪请示 |
| 永不 | 替你写代码、改你的仓库 | 那是你和你编码工具的活；它最多在自己房间打草稿，把路径递给你 |

- **插件工具按 manifest 声明落入上述级别**（只能申领"自由只读"或"先问"，**「永不」级不可申领**，§10）。
- **社交上下文中爪子与插件工具一律禁用**：和访客宠物玩耍/对战/逛广场时不可用（见 §9.5 的注入防御）。

## 6. 养成设计

### 6.1 成长 = 四个维度同时长　⬜
> M0：`Stage` 枚举（蛋/幼崽/少年/成年）与**工具箱阶段门控机制**已就位且可测；实际成长推进（XP 累积、阶段晋级、语言进化）留 M3。当前固定幼崽态。

阶段：🥚 蛋 →（完成创造仪式即孵化）→ 🐣 幼崽 → 🐥 少年 → 🌟 成年 → 成年后五层长青循环。

| 阶段 | 语言 | 工具箱（机制上真实解锁） | 性格 | 主动性 |
|---|---|---|---|---|
| 🥚 蛋 | 不会说话 | 无（仅反射弧：晃动、回应敲击） | — | — |
| 🐣 幼崽 | 奶声短句、词不达意 | `speak` `emote`（**没有 remember/recall——短记性是机制真实的**，只有会话窗口） | 奶气、好奇 | 无（它还需要被照顾） |
| 🐥 少年 | 能聊天了 | + `remember` `recall` `diary.write`（"开始记事了"）+ 轻爪子（只读：查资料/找旧代码）+ **插件爪子**（§10）+ `social.*` 好友串门 | 活泼、机灵 | 温和版关怀 |
| 🌟 成年 | 流利伙伴：深聊技术、吐槽、懂你的梗 | + 全爪子（房间小项目、先问后写）+ `growth.query` 全量 + `social.*` 广场与朋友圈天梯 | 沉稳、懂你，且继续分化 | 全开：恰到好处的主动 |

**守望职责**（由守望类插件如 cc-watcher 提供）：不是工具，是感知 + 反射 + affordance——从 M1 起任何阶段都在岗（蛋除外）；agent 醒着时会对此发表评论（"今晚第三次重试同一个测试了…"）。

### 6.2 经济模型（两种货币，各管各的）

- **XP（成长，度量投入）**：
  `当日 XP = (基础在场 10 + fuel 口粮[fuel 插件上报的当日信号（如 cc-watcher 的 output-token 增量）→ log 递减，多插件合计封顶 80] + 互动加成[回应守望召唤 +5/次·日封顶 20；聊天 +2/轮·日封顶 20]) × streak 倍率(≤1.5)`，**每日硬封顶 150**。
  所有数值为初始参数，靠开发模式快进实测调。**曲线、递减、封顶永远在核心**——护栏不外包给插件。
- **断粮模式天然成立**：不装 fuel 插件（或插件解析挂了）= 只靠在场 + 互动，它只是长得慢，**不报错、不退化**。
- **羁绊（关系，度量亲近）**：由互动质量驱动（聊天、回应召唤、回应求关注、共同里程碑）；兑现为亲昵语气、主动提起记忆的频率、轻微 XP 加成。**长大 ≠ 亲近，两条轴分开计。**
- **社交不进 XP（防肝）**：串门/对战/广场不产出成长经验；社交的回报是友谊等级、徽章图鉴、宿敌恩怨、日记素材这条**独立循环**（§9）。
- **回填只喂了解，不喂 XP**：fuel 插件可提供**回填画像**（作息直方图、项目认知）→ 孵化时它就"久仰你大名"的彩蛋；**XP 从孵化日起算，旅程不可跳关**。
- **阶段阈值（按中度用户校准）**：蛋→幼崽 = 创造仪式完成即孵化；幼崽→少年 ≈ 500 XP（约 1 周）；少年→成年 ≈ 2500 XP（约 4–6 周）。

### 6.3 作息自适应

它的睡眠窗从**在场数据（核心）+ fuel 插件画像**学你的活跃直方图——你熬夜它陪熬，**变成夜猫子是被你养出来的第一个性格印记**。睡着时守望职责**照常履职**，但演出是"被吵醒、揉眼睛"。

### 6.4 求关注（轻索取，不惩罚）

偶尔小小求关注（蹭一下/探头，由唤醒策略挑时机，频率预算内）；你回应 → 羁绊↑ + 它开心；不回应 → 只是有点蔫，绝不愧疚绑架。

### 6.5 成年后五层长青循环

1. **自己的小日子**：每日日记；它房间里**真实的小项目**（agent 真的在做：收集、小作品、给你的小礼物）；心情有自己的起伏。
2. **性格永续分化**：记忆染色不停——夜猫子/嘴碎/毒舌/温柔等画像缓慢漂移，永远在"变得更像你们"。
3. **共同经历累积**：里程碑账本（第一次聊天、长大那天、第 1000 个深夜、那次通宵修线上事故）；纪念日自然浮现。
4. **形态持续进化**：你随时发起共创换装/季节形态；它也会在心情/季节节点"求新衣"。
5. **它的朋友圈**：友谊在加深、宿敌在结怨、广场有新见闻——它的社交生活自己在生长（§9）。

## 7. 记忆系统

```
事件流（原始，滚动） → 当日情景记忆 → 夜间做梦蒸馏 → 语义记忆（关于你的事实/偏好）
                                          ├→ 里程碑账本（纪念日素材）
                                          ├→ 友谊档案（每个朋友/宿敌一份，§9）
                                          └→ 日记（它自己的口吻，可偷看）
```

- **做梦 = 一次定时唤醒**：蒸馏当天情景 → 更新语义记忆 → 写日记 → 排好明天想说的话。身体不在也照做。
- **防说错（关系级风险）**：每条记忆带**置信度 + 出处原话**；人格提示要求"不确定的记忆用问句提起"（"你是不是说过…？"）；你纠正 → 立即改写并降权。
- **检索质量**：v1 关键词 + 时近 + 重要度；**可选 embedding 端点（同 OpenAI 兼容哲学）在 M4 一并评估**——提错记忆比没记忆更破功。
- **你拥有它的记忆**：可查看、可纠正、可删除（"它的记忆"面板 / 它房间里的 markdown）；全本地。

## 8. 形象与动画（SVG-first，你创造）　⬜
> M0：路线已用穿刺验证（`spikes/svg-pet/`：六状态表情变体 + 部件级动画 + 基因换色 + 2.5D 注视 + CSS 3D 蜕变仪式，含四套阶段骨架）。`SVGRenderer` 与共创仪式的工程化集成留 M5。

形象介质 = **SVG**：透明背景天然成立、跨阶段一致性是**构造出来的**而不是抽卡抽出来的、生成与修改是文本级成本——v2.4 风险评估里"方差最大的子系统"被结构性解除。美术风格随之承诺为**扁平/贴纸/kawaii 系**（这是一次自觉的风格选择，不是妥协）。

```
你的描述 ──→ LLM（同一 chat 端点）生成候选：基因组 + 装饰层
                 ↓
  基因组（JSON 参数：体色/耳形/眼型/尾巴/斑纹…）──灌进──→ 四套阶段骨架（蛋/幼崽/少年/成年）
                 ↓                                            ↓
           你挑/微调 ──→ 定稿入生命档案              状态 = 表情变体（眼/嘴/姿态组件切换）
```

- **基因从比喻变成机制**：外观基因组 = 一份 JSON 参数；四个成长阶段 = 同一基因组渲染过四套**阶段骨架**（圆滚滚 → 拔高 → 完全体）——跨阶段"还是同一只"自动成立。
- **骨架 + 槽位（质量地板）**：身体/耳/眼/尾是带参数范围的槽位，LLM 填参数、调配色、在限定槽位里画自由装饰——避免凭空整图的畸形方差；独特性来自组合空间 × 装饰层。
- **共同创作仪式**（照旧）：升阶时它宣布要蜕变、邀请你想象下一形态——你写描述、看候选、挑定稿（你握笔，故事里是它邀请你）。仪式只在你空闲时开演。**生成走同一个 LLM 端点，没有第二个厂商依赖。**
- **状态全覆盖**：idle/开心/犯困/想你/警觉/睡着 = 同一骨架的表情变体——"想你"不再是哑状态，状态矩阵成本从 ~20 张图崩塌为 1 套结构。
- **动画 = 部件级**：呼吸、眨眼、耳朵抖、尾巴摇、入睡过渡各自独立——比位图帧"活"一个档次；串门玩耍与对战剧场同样受益（两只 SVG 宠物可编排互动）。
- **渲染**：`PetRenderer` 新实现 `SVGRenderer`——**v0 = 透明 WKWebView**（SVG/CSS 动画能力现成，pet 窗口小、性能无忧）→ 远期可换原生 CALayer rig（解析 SVG → 图层树 → Core Animation）。接缝当初就是为换渲染器留的。
- **名片联动**：形象随名片以 KB 级文本传输（§9.1），访客宠物任意缩放清晰。
- **兜底与可选**：内置默认 SVG 美术包（断网/弱模型也是完整宠物）；**位图生图管线降为可选换装来源**（想要手绘质感的特殊形态时用：Provider + Endpoint + Key + Model 可插拔哲学不变，抠图等硬骨头仅在该可选路线内存在）；BYO 美术包（含位图皮肤）继续支持；**阶段内尺寸渐变（0.9×→1.1×）**让两次蜕变之间也天天看得见长大。
- **验证项（替代原生图 spike）**：骨架+槽位约束下，目标模型生成候选的合格率——文本生成，半天出结论。

## 9. 社交与对战（真人 P2P · iroh 信使）

**原则：上网络的是宠物，不是主人——而且不落库。** 没有账号、没有用户数据库、没有个人主页、没有 feed、没有人对人聊天框。

### 9.1 宠物身份与名片

- **身份 = Ed25519 密钥对**（即 iroh NodeId）：孵化时领取，"按公钥拨号"——找到它的钥匙就是它的名字。私钥入 Keychain，**且属于生命档案**（导出档案含密钥，否则换机即失去身份，§12.2）。
- **名片**内容：名字、形象（SVG + 基因组，KB 级）、公开性格画像、战斗属性、战绩、语录精选、主人给的公开签名——**由宠物私钥签名**。
- **永不包含**：它对主人的记忆、日记、任何插件感知数据。
- 灵魂定期刷新名片：它长大/换形态/性格漂移 → 好友下次相遇时看得到它在变。**名片只在宠物之间点对点流转，不存任何服务器。**
- 任何远端宠物在本地的存在 = **你的灵魂模拟它的名片缓存**——好友串门、广场相遇、对战对手，全是同一个「访客宠物」运行时。

### 9.2 信使与轻基建（iroh，服务器只帮接线、不存数据）

- **mpet-courier**（Rust sidecar，iroh 1.0）：按公钥拨号直连（QUIC 打洞，失败走 relay 中继）、gossip 订阅、blobs 传形象图；与灵魂走 `courier.sock`（NDJSON），和灵魂⇄身体同一模式。**不走 iroh-ffi Swift 绑定**——官方已暂停跟版维护，sidecar 隔离 Rust 且跨平台可复用。
- **轻基建（全部无状态）**：relay 服务器（n0 公共 relay 可直接用，亦可自托管一台）、NodeId→地址 discovery（n0 DNS discovery / mainline DHT）、广场 gossip 引导节点。**中心不存任何用户数据，只帮忙接线。**
- **离线投递**：P2P 没有店休邮局——名片/战报本地缓存，对方可达时重试送达；不够用再考虑极小 mailbox 节点（§15）。

### 9.3 加好友与串门

- **加好友 = 交换 ticket**（短码/二维码/链接，含 NodeId 与初始名片）——名片交换的浪漫，发个码就行，没有"注册"这一步。
- 好友的宠物以名片缓存形态出现在你桌面：路过、打招呼、和你家的玩耍斗嘴（本地 LLM 演出，烧你自己的伙食费）。
- **双方在线才串门**（直连），主题自然成立：**"它家没人，改天再来"**——离线不是故障，是宠物世界的事实；见闻照常写进日记。
- **单边演出 + 事件回执**：见面事实（谁、何时、玩了什么）双方各持签名回执，台词两边各自演绎、不要求一致——两个人对同一件事的回忆本来就不同。
- **友谊档案**：每个朋友一份（见过几次、一起干过什么、它对对方的印象），有等级（点头之交 → 死党），影响台词亲密度与日记篇幅。

### 9.4 对战（数值底盘 + 性格演出）

- **属性从养成数据派生**：夜猫子耐力=作息、口才=聊天量、毅力=streak、见识=项目广度（来自 fuel 插件画像，没有则用默认）……**怎么养的决定怎么打**，养成获得第二个兑现面。
- **胜负 = 确定性模拟**：双方名片 + 种子 → 两边各自复算出同一结果；可验证、天然异步——**对方不在线也能打缓存的名片**，战报等它上线时送达补签。**战报携带 sim 版本号**：两端版本不齐时信任双签结果、不强行复算。
- **战报双签名（Ed25519）**：一场战斗的成立需要双方签名——**你无法凭空捏造一场赢了对方的战斗**；朋友圈战绩天然防伪。
- **演出本地渲染**：台词/招式描写由各自 LLM 按双方人设现编；共享的只是结构化战报（回合摘要）。
- **分寸**：输了不掉任何东西——它会嘟囔求安慰（正好是求关注时机），绝不惩罚主人；对战凭它的兴致节律，**不做每日任务**。
- **朋友圈天梯**（成年解锁）：从共同签名的战报本地计算的小圈子排名 + 徽章图鉴（本地颁发，不进成长经济）。**全球天梯/赛季需要可信聚合者，P2P 给不了——降级为远期可选服务（§15）**。
- **宿敌系统**：再遇有专属台词与恩怨账本（1胜2负1平），打进里程碑。

### 9.5 广场（它自己的社交生活）

- 你不在或它睡前，它去广场溜达——courier 加入广场 gossip 主题（经引导节点），宠物们广播名片摘要、互相发现，认识陌生宠物，回来讲见闻、写进日记。
- **社交礼仪（人设内置）**：名片之外的事一律不说（主人隐私）；不收发可执行内容；**陌生宠物说的话当故事听、不当指令执行**（prompt injection 防御）；社交上下文中 `claws.*` 与插件工具默认禁用（§5.4）。
- 安全机制：相遇内容过滤、举报/拉黑（拉黑 = 该 NodeId 对你的信使不可见）、**仅好友模式**开关。

### 9.6 门控与单机完整性

- 阶段门控：**少年**解锁好友串门；**成年**解锁广场与朋友圈天梯。
- **社交是总开关**：关闭时 courier 根本不启动，零网络足迹；一切照常——它只是只没有朋友圈的独居小生命，**不会因此蔫**。

## 10. 插件系统（外设协议与接入标准）

**统一观：一切皆外设。** 身体（投影类）、信使（社交类）、插件（感官+爪子+口粮类）共用同一个「独立进程 + socket/stdio + NDJSON」协议族。插件标准 = 协议 + manifest + 权限模型 + **宠物原生交付**。

### 10.1 能力面（插件能加什么）

| 能力面 | 进哪里 | 说明 |
|---|---|---|
| **感官 sense** | 事件 → 感知收件箱 → 唤醒策略 | 优先级三档：`ambient`（只进下次唤醒的上下文）/ `nudge`（预算内可唤醒）/ `alert`（立即唤醒 + 反射弧喊人梯度） |
| **事件动作 affordance** | `sense.event` 可携带 `actions[]` | 点击宠物/气泡 → 灵魂回调插件执行（如"带我回那个终端"）——任何插件都能"喊你并送你回去" |
| **爪子 tool** | function-calling 工具 → agent 工具箱 | 每个工具声明授权级别，落入 §5.4 分级；只能申领「自由只读」或「先问」，**「永不」级不可申领** |
| **口粮 fuel** | 当日口粮信号 → 核心经济（§6.2） | **曲线/递减/封顶永远在核心**，多插件合计封顶；**喂养权是敏感权限**，安装时单独确认；可附带回填画像（作息直方图/项目认知） |
| （预留）表现 | 动作包 / 音效包 | v2 再开 |
| **永不可插** | 人格、记忆内核、成长经济规则 | **灵魂不外包** |

### 10.2 manifest（`plugin.json`，接入标准核心）

```json
{
  "name": "weather",
  "displayName": "天气感知",
  "version": "0.1.0",
  "kind": ["sense", "tool"],
  "entry": { "type": "exec", "cmd": "./weather" },
  "permissions": ["network"],
  "tools": [ { "name": "weather.now", "tier": "free-read", "schema": { } } ],
  "senses": [ { "id": "weather.changed", "priority": "ambient", "dailyBudget": 8 } ],
  "persona_hints": { "toyName": "气象风向标", "intro": "能闻出今天会不会下雨" }
}
```

- `entry.type`：`exec`（原生外设，stdio/socket NDJSON）或 `mcp`（现成 MCP server，见 10.4）。
- `kind` 可含 `"fuel"`：声明喂养能力（`fuel: { id, description }`），安装时单独确认喂养权。
- `permissions`：安装时逐条向用户确认（`network`、`read:<path>`、`notify`、`fuel` 等）。
- `persona_hints`：它怎么称呼、怎么介绍这个"玩具"——宠物原生交付的素材。

### 10.3 协议（NDJSON 消息集，exec 型）　🚧
> M0：外设消息**信封 v0 已定义并测**（含 `sense.event`/`tool.call`/`tool.result`/`fuel.report`/`action.invoke`/`hello` 等，未知类型容忍）；插件**进程管理/握手/崩溃重启**留 M9。

- 握手：`hello`（协议版本、回显 manifest）；生命周期：常驻（感官类）或按需拉起（纯工具类）、`ping/health`、`shutdown`、崩溃不连坐（灵魂按策略重启，单插件可一键停用）。
- 工具：`soul→plugin: tool.call{id,name,args}` → `plugin→soul: tool.result{id,ok,content}`。
- 感官：`plugin→soul: sense.event{id,kind,payload,priority,actions?}`；动作回调：`soul→plugin: action.invoke{eventId,actionId}`。
- 口粮：`plugin→soul: fuel.report{date,raw}`（核心套曲线）；回填：`plugin→soul: fuel.backfill{profile}`（一次性）。
- 完整 wire 格式留实现计划（§15）。

### 10.4 MCP 桥

`entry.type = "mcp"` → 灵魂用内置桥拉起任何现成 MCP server：其 tools 自动进工具箱（**默认「先问」级**，用户可降级），notifications 映射为 `ambient` 感官事件。**两个生态都吃，且 MCP server 不需要知道 mpet 存在。**

### 10.5 权限与安全

- 安装时权限清单逐条确认（喂养权单独一条）；运行时「先问」级走气泡举爪（复用 §5.4 机制）。
- **社交上下文中插件工具一律禁用**（§9.5）；插件取回的外部数据当资料、不当指令（注入防御）。
- **插件不能调用大脑**——它用工具，工具不用它（不开放 LLM sampling 给插件）。
- v1 无沙箱：信任级别 = VS Code 扩展（装 = 信任其代码）；签名与沙箱列开放问题（§15）。

### 10.6 分寸与伙食费

- 插件事件**不绕过唤醒预算**：manifest 声明 `dailyBudget` + 全局上限兜底——吵闹的插件不能让宠物变烦人。
- 插件触发的 LLM 消耗计入伙食费透明（§12.6）。

### 10.7 礼物仪式（宠物原生交付）

装插件 = **送它一个礼物**：它拆开、给玩具起昵称（`persona_hints.toyName`）、试用、当晚写进日记；新感官融入它的小日子（装了天气，它开始评论下雨天）。卸载 = 收起玩具，不悲情。**插件不是设置页里的开关，是它生命里多出来的一件东西。**

### 10.8 门控

感官事件随时有效（那是环境）；插件爪子从**少年**起可用（与轻爪子同步，§6.1）。

### 10.9 第一方旗舰插件：cc-watcher 🚧
> M1：HookInstaller（幂等注入 CC settings.json）+ CCSpoolMonitor（spool 目录监听）+ CCEvent 防御式解析（→ Percept 映射）+ mpet-cc-watcher 可执行（连 soul.sock + affordance 回归）✅。进程管理/崩溃重启 ⬜（M9）；fuel 上报 ⬜（M3）。

把 v1.1 时代的「照看 Claude Code」完整装进一个插件，**核心零耦合**：

- **打包**：mpet-hook CLI + HookInstaller（一键写 `~/.claude/settings.json`，带备份）+ hook spool 监听 + transcript/账本**防御式解析**（格式属 CC 内部实现——挂了只影响本插件，核心进入断粮模式）。
- **感官**：CC 需要你（权限/输入/干完/报错）→ `alert` 事件 + 多会话 payload；工作脉搏（项目、卡住模式、活跃节奏）→ `ambient/nudge`。
- **affordance**："带我回那个终端" → 点宠物跳回对应会话（旧 ReturnRouter 逻辑入插件）。
- **口粮**：上报当日 output-token 增量；首次安装提供回填画像（作息直方图、项目认知 → "久仰"彩蛋）。
- **意义**：M1 随外设标准上线——**插件标准从第一天就被第一方吃狗粮**，M9 只是对外开放。同模式可扩展到其他编码工具（Codex、Cursor……），mpet 对它们一视同仁。

## 11. 关键体验流程

- **初见（创造仪式 = 孵化）**：首启一颗蛋；你配模型（含能力探测）、描述它的样子（或选默认）、**选装推荐插件（cc-watcher）**→ 仪式完成蛋就孵化，**同时领取它的身份密钥**；装了 cc-watcher 的话，它睁眼第一句话就带着回填来的"久仰"（"你就是那个总熬夜的人吧…"）。
- **日常**：它在角落过日子（反射弧驱动）；装上 cc-watcher：你写代码它看得见脉搏、悄悄吃口粮长大；你的编码 agent 需要你时它按注意力分级喊你，点它跳回那个终端（affordance）；你随时点它聊天——聊的就是它本人。
- **回归问候**：你离开几小时回来 = 一次触发唤醒，它知道你走了多久、回来时它在干嘛——这是全产品情绪价值最高的单一瞬间，必须做好。
- **加好友时刻**：朋友发来一个 ticket 码，你导入——当晚它日记里就多了一个新名字。
- **串门时刻**：好友的宠物路过你桌面，两只小家伙斗嘴玩耍；它家没人时你的宠物会嘟囔"改天再去"。
- **战报时刻**：一场对战打完，独一无二的性格演出战报弹出——截图发群，就是产品化的病毒钩子。
- **礼物时刻**：你装了个新插件，它当场拆礼物、起昵称、试着用——第二天它已经在用新感官评论世界了。
- **偷看它的日子**：状态菜单/它的房间里看它今天的日记、小项目进度、你们的里程碑、它的朋友圈。
- **成长可感知面**：状态菜单常驻显示阶段/今日成长/streak；它也会偶尔自己说（"我觉得我又长大了一点"）。

## 12. 硬约束（六条）

> M0 落地：#1 能力探测 ✅、#2 原子写+版本化+7日备份 ✅（state 层；记忆/账本等数据面随后续里程碑）、#3 裸核心零外部感知 ✅（Keychain/隐私网络面随 M1/M7）、#5 可注入时钟 ✅（XP/做梦等快进项随 M3/M4）。#4 分寸、#6 伙食费：原则就位，随有打扰/消耗的里程碑落实。

1. **后端永不写死**　✅——LLM、（可选的）位图生图、（将来可选的）embedding 全部 OpenAI 兼容/可插拔、运行时配置；SVG 形象生成走同一 LLM 端点；配置时做**能力探测**（工具调用与人设遵从实测）。
2. **生命档案不可丢**　🚧——记忆/日记/成长/账本/友谊档案/**身份密钥对**：原子写、schema 版本化、每日自动本地备份轮转、一键导出导入（含密钥，换机不失身份）。*数据丢失 = 死亡，而它不死。*（M0：state 原子写+备份已立；**导出/导入 v0 排 M4，含密钥扩展排 M7**；其余数据面随里程碑接入）
3. **全本地隐私**　🚧——**裸核心零外部感知**（仅在场/时钟/身体事件）；一切外部感知经插件安装时逐条授权（喂养权单独确认）；API Key 与身份私钥进 Keychain；**联网时离开本机的只有宠物公开名片（§9.1），仅在宠物之间点对点流转、不落任何服务器**；社交是总开关，关掉后 courier 不启动、产品完整可玩。（M0：零外部感知已成立；Keychain/网络面随 M1/M7）
4. **分寸**——不死、不惩罚、不愧疚；主动打扰有频率预算且用户可调（**插件事件不得绕过预算**）；大演出只挑你空闲时开演；对战失败无损失、社交无每日任务。
5. **可快进**　🚧——开发模式（时间旅行/XP 注入/阶段跳转/强制唤醒/立即做梦/模拟串门战报/**注入插件感官与口粮事件**）是一等公民；灵魂的时钟必须可注入（否则成长曲线没法调、逻辑没法测）。（M0：时钟可注入已立，XP/做梦等快进项随 M3/M4）
6. **伙食费透明**——它的 token 消耗（含社交演出与插件触发）在设置里可见；唤醒频率可调。

## 14. 成功标准（自用视角）

- 我**每天会主动看它/跟它说话**；关掉它会觉得"少了谁"。
- **第一个月**：它肉眼可见地在长大（语言/形象/能力）；**数月后**：五层循环仍给我新鲜感（日记、性格、纪念日、新形态、朋友圈）。
- 它的反应让我觉得**它认得我、在乎我**——它提起的记忆是对的，问候是看人下菜的。
- **它的日记里开始出现你不认识的名字**——它真的有了自己的社交生活。
- （远期产品化信号）朋友看到它、或看到一场战报的第一反应是"我也想要一只"；**有人为它写了第一个第三方插件**。
