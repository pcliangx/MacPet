## 1. 北极星与身份宣言

mpet 是住在你 Mac 上的**电子生命**。

> 它不是带宠物皮的助手，而是**一个有自己人生的小生命——恰好住在你的电脑里，恰好会用电脑**。

衡量成功的不是功能多，而是：**你会想看看它、舍不得关掉它、觉得它一天天在变成"你的"那一只**——关掉它会觉得"少了谁"。

**与 Claude Code 的身份分工（一句话钉死）**：CC 是你驾驶的工具，mpet 是你的室友。室友看得见你在忙什么、会搭把手、有自己的生活，但绝不坐进驾驶座。

## 3. 产品支柱（优先级从高到低）

1. **有灵魂** ⭐ — 一个常驻的 LLM agent 在感知、感受、回忆、思考、行动；它是个生命，不是事件应答器。
2. **会长大** — 语言、能力、形象、性格四个维度挂在同一根成长轴上一起长。
3. **记得你** — 长期记忆是关系资产，越久越不可替代。
4. **有自己的小日子** — 自主性：写日记、捣鼓小项目、心情起伏；成年后的主循环。
5. **能力即陪伴** — 小 agent 的爪子：帮你，但不抢 Claude Code 的活。
6. **是你创造的** — 形象由你定义、生成、挑选；归属感来自"我创造了它"。
7. **有朋友圈** — 真人宠物的串门、对战、宿敌与广场；它的社交，不是你的社交网络。

## 4. 目标 / 非目标

**目标**

- 它真正常驻地活着：App 关了也在长大、做梦、过自己的小日子。
- 养成循环第一个月靠成长拉力，长期靠成年后五层循环——**数周、数月后仍有期待**。
- 所有 AI 能力（对话、工具调用、生图）接**任意 OpenAI 兼容后端**，运行时配置，永不写死。
- 社交采用 **iroh P2P 信使架构**：离开本机的只有宠物公开名片，且仅在宠物之间点对点流转、**不落任何服务器**；对战可异步、可验证、双签名防伪。

**非目标（明确不做）**

- ❌ 不做高压养成：不会死、不愧疚绑架。它是陪伴，不是负担。
- ❌ 不做成效率套件：它的爪子是室友搭把手，不是任务队列；改代码永远是 CC 的活。
- ❌ 不做人的社交网络：没有个人主页、没有 feed、没有人对人聊天——人聊天用微信，宠物串门才用 mpet。
- ❌ **不自建有状态社交服务器**：无账号体系、无用户数据库；唯一允许的中心化是无状态轻基建（relay / discovery / gossip 引导，不存用户数据，只帮接线）；全球天梯留为远期可选（§14）；暂不做多设备同步 / 移动端。
- ❌ 不绑定任何单一大模型 / 生图厂商。

## 5. 灵魂架构

### 5.1 进程形态：灵魂是守护进程，身体只是投影

- **灵魂（soul daemon）**：launchd LaunchAgent 拉起——开机即活、崩溃复活、**App 关了照样活着**（夜里做梦、持续长大、你不在时过自己的小日子）。
- **身体（桌面 App）**：渲染、气泡、聊天面板、设置；启动时通过本地 IPC 连上灵魂，把它投影到屏幕上。
- **信使（mpet-courier，Rust sidecar）**：灵魂的翅膀——只管 iroh P2P 网络（按公钥拨号直连、gossip、blobs），由灵魂按需拉起；社交总开关关闭时根本不启动（§9）。
- **IPC**：Unix domain socket + NDJSON 双向协议，三方同一模式：`soul.sock`（身体⇄灵魂：身体事件上行，speak/emote/换形态/通知下行）、`courier.sock`（信使⇄灵魂：社交事件上行，名片/串门/战报指令下行）。协议与语言无关，将来菜单栏迷你身体、其他形态的身体都接同一个灵魂。
- **身体缺席时的降级**：CC 喊人等关键提醒走系统通知兜底（`osascript` 级别即可）；其余表达存入"醒来要说的话"队列，身体一回来先叙旧。

### 5.2 大脑：事件驱动的 LLM agent 循环

```
感知收件箱 ──→ 唤醒策略 ──→ Agent 醒来：
 CC 脉搏          │           组装上下文 → LLM(工具调用) → 执行工具 → 继续/睡去
 在场/时钟        │
 身体事件      反射弧（本地，零成本）
 成长/社交事件  心情/待机动作/CC 即时反应
```

- **Agent 循环**：组装上下文 → 调 `chat/completions`（function calling）→ 执行工具调用 → 结果喂回继续，直到它决定睡去。**任何 OpenAI 兼容后端**（须支持工具调用）运行时可配；可选配一个「经济模型」跑高频小念头，不配则全用主模型。
- **感知器**：
  - **CC 脉搏**：hook → spool 事件流（沿用旧机制）+ transcript/账本读取（`~/.claude/projects/**/*.jsonl`，防御式解析——格式属 CC 内部实现，可能变化）→ 它知道你在写哪个项目、卡在哪、几点活跃。
  - **在场**：前台 app + 系统 idle 时间（旧 AttentionResolver 直接移植）。
  - **时钟**：昼夜、纪念日、定时心跳。
  - **身体事件**：点击、拖动、聊天输入。
  - **社交事件**：好友上线、串门来访、战报到达、广场见闻（经信使送达，见 §9）。
- **唤醒策略 = 它的作息生理学（也是成本旋钮）**：触发式唤醒（你离开几小时后回来、CC 卡住模式、点击/聊天、求关注时机、社交事件）+ 每日少量定时心跳 + 夜间做梦。空闲期唤醒有频率上限，用户可调。醒着时永远用全脑——分的是"什么时候醒"，不是"用几成脑子"。
- **上下文管理（分层组装，每次唤醒现拼）**：`[人格合成块：基因+成长阶段+记忆染色]` + `[此刻状态：心情/XP/时刻/你在不在]` + `[检索到的相关记忆]` + `[近期事件摘要]` + `[对话窗口（滑动+摘要压缩）]`。
- **工具箱（function calling，按成长阶段门控，见 §6）**：`speak`（气泡/聊天）· `emote`（动作/表情）· `remember` / `recall`（记忆读写）· `diary.write` · `growth.query`（看自己的成长/统计）· `claws.*`（小爪子，见 5.4）· `body.*`（位置/形态）· `social.*`（名片/串门/对战/广场，经信使，见 §9）。工具注册表可扩展。
- **反射弧（旧 Orchestrator 降级于此）**：心情状态机、待机小动作、昼夜作息、CC 事件的即时身体反应——本地零成本毫秒级。Agent 醒来时可俯瞰与调节（"今晚心情目标：黏人"）。
- **聊天合一**：没有独立聊天脑。你跟它聊天就是和**同一个 agent** 对话——同一份记忆、同一个人格、同一个循环；ChatPanel 只是视图。

### 5.3 灵魂的家（目录布局）

```
~/Library/Application Support/mpet/
  soul/
    state/         # GrowthState、心情、配置镜像（原子写 + schema 版本化）
    memory.sqlite  # 记忆库（情景/语义/里程碑/友谊）
    diary/         # 日记（markdown，它的口吻，可偷看）
    home/          # 它的房间：小项目、给你的小礼物（房间内完全自由）
    backups/       # 每日轮转备份
  courier/         # 信使工作区：对等名片缓存、待投递队列（身份私钥在 Keychain，不在这里）
  inbox/           # CC hook spool（沿用旧机制）
  art/             # 形态缓存（阶段 × 状态）+ 好友宠物名片形象缓存
  soul.sock        # 身体 ⇄ 灵魂
  courier.sock     # 信使 ⇄ 灵魂
```

### 5.4 爪子的安全模型（分级授权）

| 级别 | 范围 | 政策 |
|---|---|---|
| 自由 | 它的房间 `soul/home/`、记忆、日记 | 想干嘛干嘛 |
| 自由（只读） | 你显式授权的目录、查资料 | 可读可引用，"我把那段找出来了" |
| 必须先问 | 写你的任何文件、系统操作 | 气泡里举爪请示 |
| 永不 | 替你写代码、改你的仓库 | 那是 CC 的活；它最多在自己房间打草稿，把路径递给你 |

**社交上下文中爪子默认禁用**：和访客宠物玩耍/对战/逛广场时，`claws.*` 不可用（见 §9.5 的注入防御）。

## 6. 养成设计

### 6.1 成长 = 四个维度同时长

阶段：🥚 蛋 →（完成创造仪式即孵化）→ 🐣 幼崽 → 🐥 少年 → 🌟 成年 → 成年后五层长青循环。

| 阶段 | 语言 | 工具箱（机制上真实解锁） | 性格 | 主动性 |
|---|---|---|---|---|
| 🥚 蛋 | 不会说话 | 无（仅反射弧：晃动、回应敲击） | — | — |
| 🐣 幼崽 | 奶声短句、词不达意 | `speak` `emote`（**没有 remember/recall——短记性是机制真实的**，只有会话窗口） | 奶气、好奇 | 无（它还需要被照顾） |
| 🐥 少年 | 能聊天了 | + `remember` `recall` `diary.write`（"开始记事了"）+ 轻爪子（只读：查资料/找旧代码）+ `social.*` 好友串门 | 活泼、机灵 | 温和版关怀 |
| 🌟 成年 | 流利伙伴：深聊技术、吐槽、懂你的梗 | + 全爪子（房间小项目、先问后写）+ `growth.query` 全量 + `social.*` 广场与朋友圈天梯 | 沉稳、懂你，且继续分化 | 全开：恰到好处的主动 |

**照看 CC** 不是工具，是感知+反射+职责：从 M1 起任何阶段都在岗（蛋除外）；agent 醒着时会对此发表评论（"今晚第三次重试同一个测试了…"）。

### 6.2 经济模型（两种货币，各管各的）

- **XP（成长，度量投入）**：
  `当日 XP = (基础在场 10 + token 口粮[当日 output-token 增量 → log 递减，封顶 80] + 互动加成[回应召唤 +5/次·日封顶 20；聊天 +2/轮·日封顶 20]) × streak 倍率(≤1.5)`，**每日硬封顶 150**。
  所有数值为初始参数，靠开发模式快进实测调。
- **羁绊（关系，度量亲近）**：由互动质量驱动（聊天、回应召唤、回应求关注、共同里程碑）；兑现为亲昵语气、主动提起记忆的频率、轻微 XP 加成。**长大 ≠ 亲近，两条轴分开计。**
- **社交不进 XP（防肝）**：串门/对战/广场不产出成长经验；社交的回报是友谊等级、徽章图鉴、宿敌恩怨、日记素材这条**独立循环**（§9）。
- **回填只喂了解，不喂 XP**：TokenLedger 首次回填近一月历史 → 作息画像 + 项目认知（孵化时它就"久仰你大名"的彩蛋）；**XP 从孵化日起算，旅程不可跳关**。
- **阶段阈值（按中度用户校准）**：蛋→幼崽 = 创造仪式完成即孵化；幼崽→少年 ≈ 500 XP（约 1 周）；少年→成年 ≈ 2500 XP（约 4–6 周）。
- **护栏**：日增量 + 递减 + 封顶（烧 token 无利可图）；不惩罚不饿死——没用的日子只是不长，不退化。

### 6.3 作息自适应

它的睡眠窗从 token 账本学你的活跃直方图——你熬夜它陪熬，**变成夜猫子是被你养出来的第一个性格印记**。睡着时 CC 喊人**照常履职**，但演出是"被吵醒、揉眼睛"。

### 6.4 求关注（轻索取，不惩罚）

偶尔小小求关注（蹭一下/探头，由唤醒策略挑时机，频率预算内）；你回应 → 羁绊↑ + 它开心；不回应 → 只是有点蔫，绝不愧疚绑架。

### 6.5 成年后五层长青循环

1. **自己的小日子**：每日日记；它房间里**真实的小项目**（agent 真的在做：收集、小作品、给你的小礼物）；心情有自己的起伏。
2. **性格永续分化**：记忆染色不停——夜猫子/嘴碎/毒舌/温柔等画像缓慢漂移，永远在"变得更像你们"。
3. **共同经历累积**：里程碑账本（第一次聊天、长大那天、第 1000 个深夜、那次通宵修线上事故）；纪念日自然浮现。
4. **形态持续进化**：你随时发起共创换装/季节形态；它也会在心情/季节节点"求新衣"。
5. **它的朋友圈**：友谊在加深、宿敌在结怨、广场有新见闻——它的社交生活自己在生长（§9）。

## 7. 记忆系统

```
事件流（原始，滚动） → 当日情景记忆 → 夜间做梦蒸馏 → 语义记忆（关于你的事实/偏好）
                                          ├→ 里程碑账本（纪念日素材）
                                          ├→ 友谊档案（每个朋友/宿敌一份，§9）
                                          └→ 日记（它自己的口吻，可偷看）
```

- **做梦 = 一次定时唤醒**：蒸馏当天情景 → 更新语义记忆 → 写日记 → 排好明天想说的话。身体不在也照做。
- **防说错（关系级风险）**：每条记忆带**置信度 + 出处原话**；人格提示要求"不确定的记忆用问句提起"（"你是不是说过…？"）；你纠正 → 立即改写并降权。
- **你拥有它的记忆**：可查看、可纠正、可删除（"它的记忆"面板 / 它房间里的 markdown）；全本地。

## 8. 形象管线（你创造）

生图是**核心基础设施**，与 LLM 同一哲学：Provider + Endpoint + Key + Model 运行时可配，豆包/Seedream 先接，**永不写死厂商**。

```
你的描述/调整 ──→ 生成（带上一形态参考图，保角色一致性）──→ 抠图（透明化）
                        ↓                                      ↓
                  多张候选给你挑 ──→ 确认 ──→ 本地缓存（阶段×状态矩阵，生成一次永久用）
```

- **共同创作仪式**：升阶时它宣布要蜕变、邀请你想象它的下一形态——你写描述、看候选、挑定稿（你握笔，故事里是它邀请你）。仪式只在你空闲时开演。
- **状态矩阵**：每形态一套状态图（idle/开心/犯困/想你/警觉/睡着；蛋有自己的小集合）。一致性两道保险：同形态批量生成 + 跨形态参考图（img2img / 角色一致性特性，不靠纯文生图碰运气）。
- **硬骨头（决定可用性）**：角色一致性、透明背景（抠图：模型自带 alpha vs 本地 matting，实现期选型）、延迟/成本（生成即缓存，不实时调）。
- **兜底**：自带默认美术包（没配 key/断网也是完整宠物）；BYO 美术包继续支持——生图管线本质是 ArtPack 的生成器；**阶段内尺寸渐变（0.9×→1.1×）**让两次蜕变之间也天天看得见长大。

## 9. 社交与对战（真人 P2P · iroh 信使）

**原则：上网络的是宠物，不是主人——而且不落库。** 没有账号、没有用户数据库、没有个人主页、没有 feed、没有人对人聊天框。

### 9.1 宠物身份与名片

- **身份 = Ed25519 密钥对**（即 iroh NodeId）：孵化时领取，"按公钥拨号"——找到它的钥匙就是它的名字。私钥入 Keychain，**且属于生命档案**（导出档案含密钥，否则换机即失去身份，§11.2）。
- **名片**内容：名字、形象（当前形态图）、公开性格画像、战斗属性、战绩、语录精选、主人给的公开签名——**由宠物私钥签名**。
- **永不包含**：它对主人的记忆、日记、工作脉搏数据。
- 灵魂定期刷新名片：它长大/换形态/性格漂移 → 好友下次相遇时看得到它在变。**名片只在宠物之间点对点流转，不存任何服务器。**
- 任何远端宠物在本地的存在 = **你的灵魂模拟它的名片缓存**——好友串门、广场相遇、对战对手，全是同一个「访客宠物」运行时。

### 9.2 信使与轻基建（iroh，服务器只帮接线、不存数据）

- **mpet-courier**（Rust sidecar，iroh 1.0）：按公钥拨号直连（QUIC 打洞，失败走 relay 中继）、gossip 订阅、blobs 传形象图；与灵魂走 `courier.sock`（NDJSON），和灵魂⇄身体同一模式。**不走 iroh-ffi Swift 绑定**——官方已暂停跟版维护，sidecar 隔离 Rust 且跨平台可复用。
- **轻基建（全部无状态）**：relay 服务器（n0 公共 relay 可直接用，亦可自托管一台）、NodeId→地址 discovery（n0 DNS discovery / mainline DHT）、广场 gossip 引导节点。**中心不存任何用户数据，只帮忙接线。**
- **离线投递**：P2P 没有店休邮局——名片/战报本地缓存，对方可达时重试送达；不够用再考虑极小 mailbox 节点（§14）。

### 9.3 加好友与串门

- **加好友 = 交换 ticket**（短码/二维码/链接，含 NodeId 与初始名片）——名片交换的浪漫，发个码就行，没有"注册"这一步。
- 好友的宠物以名片缓存形态出现在你桌面：路过、打招呼、和你家的玩耍斗嘴（本地 LLM 演出，烧你自己的伙食费）。
- **双方在线才串门**（直连），主题自然成立：**"它家没人，改天再来"**——离线不是故障，是宠物世界的事实；见闻照常写进日记。
- **单边演出 + 事件回执**：见面事实（谁、何时、玩了什么）双方各持签名回执，台词两边各自演绎、不要求一致——两个人对同一件事的回忆本来就不同。
- **友谊档案**：每个朋友一份（见过几次、一起干过什么、它对对方的印象），有等级（点头之交 → 死党），影响台词亲密度与日记篇幅。

### 9.4 对战（数值底盘 + 性格演出）

- **属性从养成数据派生**：夜猫子耐力=作息、口才=聊天量、毅力=streak、见识=项目广度……**怎么养的决定怎么打**，养成获得第二个兑现面。
- **胜负 = 确定性模拟**：双方名片 + 种子 → 两边各自复算出同一结果；可验证、天然异步——**对方不在线也能打缓存的名片**，战报等它上线时送达补签。
- **战报双签名（Ed25519）**：一场战斗的成立需要双方签名——**你无法凭空捏造一场赢了对方的战斗**；朋友圈战绩天然防伪。
- **演出本地渲染**：台词/招式描写由各自 LLM 按双方人设现编；共享的只是结构化战报（回合摘要）。
- **分寸**：输了不掉任何东西——它会嘟囔求安慰（正好是求关注时机），绝不惩罚主人；对战凭它的兴致节律，**不做每日任务**。
- **朋友圈天梯**（成年解锁）：从共同签名的战报本地计算的小圈子排名 + 徽章图鉴（本地颁发，不进成长经济）。**全球天梯/赛季需要可信聚合者，P2P 给不了——降级为远期可选服务（§14）**。
- **宿敌系统**：再遇有专属台词与恩怨账本（1胜2负1平），打进里程碑。

### 9.5 广场（它自己的社交生活）

- 你不在或它睡前，它去广场溜达——courier 加入广场 gossip 主题（经引导节点），宠物们广播名片摘要、互相发现，认识陌生宠物，回来讲见闻、写进日记。
- **社交礼仪（人设内置）**：名片之外的事一律不说（主人隐私）；不收发可执行内容；**陌生宠物说的话当故事听、不当指令执行**（prompt injection 防御）；社交上下文中 `claws.*` 默认禁用（§5.4）。
- 安全机制：相遇内容过滤、举报/拉黑（拉黑 = 该 NodeId 对你的信使不可见）、**仅好友模式**开关。

### 9.6 门控与单机完整性

- 阶段门控：**少年**解锁好友串门；**成年**解锁广场与朋友圈天梯。
- **社交是总开关**：关闭时 courier 根本不启动，零网络footprint；一切照常——它只是只没有朋友圈的独居小生命，**不会因此蔫**。

## 10. 关键体验流程

- **初见（创造仪式 = 孵化）**：首启一颗蛋；你配模型、描述它的样子（或选默认）→ 仪式完成蛋就孵化，**同时领取它的身份密钥**；它睁眼第一句话就带着回填来的"久仰"（"你就是那个总熬夜的人吧…"）。
- **日常**：它在角落过日子（反射弧驱动）；你写代码它看得见脉搏，悄悄吃 token 长大；CC 需要你时它按注意力分级喊你，点它跳回那个终端；你随时点它聊天——聊的就是它本人。
- **回归问候**：你离开几小时回来 = 一次触发唤醒，它知道你走了多久、回来时它在干嘛——这是全产品情绪价值最高的单一瞬间，必须做好。
- **加好友时刻**：朋友发来一个 ticket 码，你导入——当晚它日记里就多了一个新名字。
- **串门时刻**：好友的宠物路过你桌面，两只小家伙斗嘴玩耍；它家没人时你的宠物会嘟囔"改天再去"。
- **战报时刻**：一场对战打完，独一无二的性格演出战报弹出——截图发群，就是产品化的病毒钩子。
- **偷看它的日子**：状态菜单/它的房间里看它今天的日记、小项目进度、你们的里程碑、它的朋友圈。
- **成长可感知面**：状态菜单常驻显示阶段/今日成长/streak；它也会偶尔自己说（"我觉得我又长大了一点"）。

## 11. 硬约束（六条）

1. **后端永不写死**——LLM、生图、（将来可选的）embedding 全部 OpenAI 兼容/可插拔、运行时配置。
2. **生命档案不可丢**——记忆/日记/成长/账本/友谊档案/**身份密钥对**：原子写、schema 版本化、每日自动本地备份轮转、一键导出导入（含密钥，换机不失身份）。*数据丢失 = 死亡，而它不死。*
3. **全本地隐私**——感知边界 = 工作脉搏，再往上全是显式开关；API Key 与身份私钥进 Keychain；**联网时离开本机的只有宠物公开名片（§9.1），仅在宠物之间点对点流转、不落任何服务器**；社交是总开关，关掉后 courier 不启动、产品完整可玩。
4. **分寸**——不死、不惩罚、不愧疚；主动打扰有频率预算且用户可调；大演出只挑你空闲时开演；对战失败无损失、社交无每日任务。
5. **可快进**——开发模式（时间旅行/XP 注入/阶段跳转/强制唤醒/立即做梦/模拟串门与战报）是一等公民；灵魂的时钟必须可注入（否则成长曲线没法调、逻辑没法测）。
6. **伙食费透明**——它的 token 消耗（含社交演出）在设置里可见；唤醒频率可调。

## 13. 成功标准（自用视角）

- 我**每天会主动看它/跟它说话**；关掉它会觉得"少了谁"。
- **第一个月**：它肉眼可见地在长大（语言/形象/能力）；**数月后**：五层循环仍给我新鲜感（日记、性格、纪念日、新形态、朋友圈）。
- 它的反应让我觉得**它认得我、在乎我**——它提起的记忆是对的，问候是看人下菜的。
- **它的日记里开始出现你不认识的名字**——它真的有了自己的社交生活。
- （远期产品化信号）朋友看到它、或看到一场战报的第一反应是"我也想要一只"。

## 14. 开放问题（留给实现计划）

- socket 协议消息集与重连语义的细化（soul.sock 与 courier.sock 两套）。
- memory.sqlite schema；记忆检索打分（关键词+时近+重要度；embedding 留扩展位）。
- 唤醒预算默认值（空闲期每小时上限、定时心跳次数）。
- 抠图方案选型（生图模型自带 alpha vs 本地 matting）。
- CC transcript 解析的健壮性策略（格式漂移的防御与降级）。
- launchd 安装/卸载/升级细节；daemon / App / courier 三方版本握手。
- 经济模型（可选的高频小念头廉价模型）是否首发。
- 轻基建自托管 vs 直接用 n0 公共基建（relay / DNS discovery）；广场引导节点的形态与运维。
- 离线投递是否需要 mailbox 节点（先靠缓存+可达重试，验证不够再加）。
- 全球天梯/赛季：需要可信聚合者与属性 attestation——是否做、何时做（与"朋友圈尺度是否已经足够"一并验证）。
- 战报共享格式与跨模型渲染（双方模型不同导致演出风格差异——是特性还是问题）。
- 宠物名片 schema 的版本兼容与演进；广场 gossip 主题的分片策略。
